Section 1: Overview

Originator:Minnesota Department of Education

Title: School Program Locations, Minnesota, SY2025-26

Abstract: This dataset attempts to represent the point locations of every educational program in the state of Minnesota that is currently operational and reporting to the Minnesota Department of Education. It can be used to identify schools, various individual school programs, school districts (by office location), colleges, and libraries, among other programs. Please note that not all school programs are statutorily required to report, and many types of programs can be reported at any time of the year, so this dataset is by nature an incomplete snapshot in time.

Maintenance of these locations is a result of an ongoing project to identify current school program locations where Food and Nutrition Services Office (FNS) programs are utilized. The FNS Office is in the Minnesota Department of Education (MDE). GIS staff at MDE maintain the dataset using school program and physical addresses provided by local education authorities (LEAs) for an MDE database called "MDE ORG". MDE GIS staff track weekly changes to program locations, along with comprehensive reviews each summer. All records have been reviewed for accuracy or edited at least once since January 1, 2020.

Note that there may remain errors due to the number of program locations and inconsistency in reporting from LEAs and other organizations. Some organization types (such as colleges and treatment programs) are not subject to annual reporting requirements, so various records included in this file may in fact be inactive or inaccurately located.

Note that multiple programs may occur at the same location and are represented as separate records. For example, an elementary and secondary school may be in the same building, but each has a separate record in the data layer. Users may leverage the "CLASS" and "ORGTYPE" attributes to filter and sort records according to their needs. In general, records at the same physical address will be located at the same coordinates.

This data is also available in CSV format. For that format only, OBJECTID and Shape columns are removed, and the Shape column is replaced by Latitude and Longitude columns.

Purpose: The purpose of this layer is to provide location information for educational programs operating within the state of Minnesota. These locations are used by MDE to help identify where appropriate funds should be used for that school year. Note that since multiple programs may operate within the same building (at the same address), this dataset is not particularly useful as a "school building" dataset. Some, but not all, public school buildings are represented as discrete records in the related "struc-school-buildings" dataset.

Please also note that all addresses shared in this layer are primarily physical addresses, not mailing addresses. While in many cases those are the same, in many others they are not. Organizations provide addresses specific for mailing purposes, which can be obtained by looking up the organization in the MDE ORG database.

Section 2: Data Quality

Attribute Accuracy: Several checks are performed manually, such as comparing addresses in the GIS data against the physical addresses in MDE ORG. Other attributes, such as GRADERANGE, CLASS, and WEB_URL, were derived from MDE ORG via SQL queries in order to include the most recent data reported to MDE by Minnesota School Districts, Charter Schools, and other educational organizations.

Logical Consistency: Not applicable

Completeness: Contains every educational program in the state of Minnesota that is currently operational and reporting to the Minnesota Department of Education.

Horizontal Positional Accuracy: Each public school district performs annual checks of MDE's databases and maps for accuracy of school names, district boundaries and public school building locations. Locations are modified through a weekly process of reviewing updated program physical addresses against various sources (aerial photography, geocoding).

All other school locations (charters, nonpublic, post-secondary, etc.) are synched with MDE-ORG database records at MDE and spatially located using address matching techniques and most current aerial photography. All locations have been reviewed for accuracy and/or edited at least once since January 1, 2020, and the date of the most recent update is tracked in the UPDATEDT column.

Each review uses various sources to determine location accuracy, such as address information, online mapping platforms, geocoding, aerial photography, county parcel and address data, commercial and publicly provided basemap layers, and reviews of education organization websites. Most records have been verified against multiple sources when determining the horizontal position.

In determining the precise location for each record, the following decision criteria are used:
1. Programs using the same physical address are co-located. Note that some large buildings are referenced with multiple building numbers in the address, thereby creating situations where more than one set of point records will be on that building. Also note that some specific addresses are used to reference multiple buildings, such as on college campuses.
2. A program "point" is placed within a parcel with the same address number (if available in parcel data shared with the state). If no such parcel exists, the remaining rules are still followed.
3. A point is placed on top of the building footprint (if visible).
4. If an address point record from the city or county is available, it is "snapped" to that location when the record is reviewed, but only if it meets the other rules. For example, address points are often placed on driveways instead of buildings, and in these cases the building footprint is prioritized for the point location.
5. If possible without violating the other rules, a point is placed closest to the street with a matching name to the address.

Since it is impossible to meet all these rules for every record, some subjectivity exists in the placement of points.

Details on selected individual columns:

orgid: A unique identifier for the program/location, used by MDE in internal databases and typically referred to as 'stateOrganizationId'. This is a numeric representation of the 'orgtype', 'orgnumber', and 'schnumber' columns, followed at the end by '000'.
example: 1 0001 002 000 is Rippleside Elementary
example: 89 8016 010 000 is Aitkin Public Library
(Spaces are used in the above examples for readability and are not used in the data itself.)

orgtype: MDE Organization Classification, derived from MDE ORG database's "districtType" column (but applies to organizations outside of school districts). Left-padded with a zero for a 2-character representation. Code reference:
01 - Independent districts

07 - Charter school

31 - Nonpublic School in Independent Districts
32 - Nonpublic School in Common Districts
33 - Nonpublic School in Special Districts
34 - Bureau of Indian Affairs (BIA) Tribal Schools
36 - NonPublic Archdiocese
50 - Miscellaneous Cooperatives
51 - Vocational Cooperatives
52 - Special Education Cooperatives
60 - Correctional Facilities
61 - Education Districts

80 - Adult Basic Education (ABE)

83 - Service Cooperative-ECSU
86 - Library Minitex Location
88 - Library Network/Cooperative
89 - Library

orgnumber: MDE school district (or other organization) number, derived from MDE ORG database. Left-padded with zeroes for a 4-character representation.

schnumber: MDE school number (or other member identifier), derived from MDE ORG database. Left-padded with zeroes for a 3-character representation.

formid: A concatenation of 'orgnumber', 'orgtype', and SCHNUMBER, separated by dashes. This is a "formatted ID" that can be used to look up the location details in the MDE ORG reporting pages.

gisname: School name stored in the spatial database. Once a school is established in the database, this is generally a simplification or a more detailed description of the school, depending on which makes the record more unique. Schools that have changed names since being established in the spatial database are not always updated in this column, although annual synching is attempted. In some cases this name is an abbreviation used for ease in map labeling.

mdename: "Preferred" school name stored in the MDE ORG database. This data is updated more frequently but is complementary to 'schname'. For example, 'orgid' 10912051000 has a preferred name of merely "ESY" in MDE ORG, but the spatial database provides a more unique description as "Milaca ESY".

altname: a column for storage of alternate names. Particularly useful when labeling school district offices with the short name for the school district, or libraries with a shorter name.

gisaddr: a concatenation of the 'address', 'city', and 'zip' columns from the spatial database (with a static "MN" inserted for geocoding/mailing purposes). Much like 'gisname' vs. 'mdename', the address data stored with the spatial database may be better, worse, or just different than what is stored in MDE ORG.

mdeaddr: a similar concatenation of address fields, but from the MDE ORG database (using physical address information where available). Note that mailing addresses are stored separately in the MDE ORG database and are NOT reflected in this dataset.

pubpriv: a classification intended to demonstrate whether a program is public, or nonpublic/private
Private or Nonpublic - All locations with orgtype IN (24,31,32,33,36)
Public - All other locations (orgtype NOT IN (24,31,32,33,36))

loctype: a classification intended to demonstrate whether a location is the "Head" (Parent) record, such as a school district or charter office, or a "Member" (Child) record, such as an individual school program location. These designations reflect the 'schnumber' column (000 vs non-zero), serving as a vehicle for more general categorization.

Some common uses of the above columns include:

1. School district offices have "000000" at the end of the 'orgid' column because they represent the "Head" record for schools within the district. Examples include Minneapolis (30001000000), Fridley (10013000000) or St. Paul (10625000000). Public school district offices can be isolated with a query of: orgtype IN ('01', '02', '03') and schnumber = '000'. Note that using point locations to represent school districts can help overcome significant cartographic challenges in small scall mapping of districts.

2. Charter schools - Charters are their own local education authority (LEA), operating much like a district without any attendance boundaries. Therefore, there is a "Head" location for each licensed charter carrying the "000000" at the end of the 'orgid' column. There may be 2, 3, 4 or more locations, depending on the charter, but there will always be one record for each charter organization and at least one record for a campus/school/site. The charters may also be housed on a single location or in several locations. To query a unique list of all Charter school organizations, query orgtype = '07' And schnumber = '000'.

If you want all charter school campus locations, even when there are multiples per charter, then query orgtype = '07' AND schnumber <> '000'

graderange: Grade levels pulled from the MDE ORG database. There is wide variation within this column and it should not be used to "classify" different types of schools. (See the 'class' column.) Some examples:
EC - Early Childhood
PK - Pre-Kindergarten
KG - Kindergarten
1 to 12 - Grades 1 to 12

Note that because nonpublic schools are not classified with this column, there is no way to discretely identify "nonpublic elementary schools".

magnet: Magnet school (YES/NO)

locdistid: Unique MDE ORG number for the school district within which the program is located. This is derived by location, and for public, non-charter schools, will typically reflect the 'orgid' of the school district, although there are exceptions. This identifier can be useful for queries such as, "how many nonpublic or charter schools operate within district X?"

locdistname: School district name within which the program is located; like 'locdistid', this is derived via location.

countycode: Minnesota standard, 5-digit County FIPS codes, derived from where the program site is located.

countyname: County name, derived from where the program site is located.

createdt: date program location was entered in the database (system generated).

updatedt: date program location was last updated in the database (system generated).

fed_schnum: National Center for Educational Statistics (NCES) school identification number. The first 2 digits are Minnesota's state number (27). The next 5 digits are the District school designation. This will be the same for each school in that district. The final 5 numbers are the school identifier. Note that these identifiers are NOT equivalent to MDE's organization identifiers.

Example: Aitkin School District
District (Office record): 01 0001 000 000 - fed_schnum=27 02760 (2702760)
Aitkin Secondary: 01 0001 001 000 - fed_schnum=27 02760 00057 (270276000057)
Rippleside Elementary: 01 0001 002 000 - fed_schnum=27 02760 00059 (270276000059)

City Academy Charter (District): 07 4000 000 000 - fed_schnum=27 08955 (2708955)
City Academy Charter (School): 07 4000 010 000 - fed_schnum=27 08955 00016 (270895500016)

web_url: website address for the school/district, derived from MDE ORG database

year: School year of data.
